01 — The Nine Beats
Video anatomy — long-form
Nine beats, fixed order. Beats 5b, 6 and 7 are conditional — included whenever the question genuinely has them, cut when it doesn't. A full-featured DEBRIEF runs 8–14 min; a lean one 5–7 min.
1 Q
2 ⏸
3 READ
4 CONCEPTS
5 PATH A
5b PATHS B/C
6 HACKS
7 TRAP
8 LOCK
1 QUESTION · 0–15s
Cold open — the question card, fully formed, in the correct format variant (ch. 02–03). One spoken line frames the stakes.
2 PAUSE · 5s
"Pause and attempt it" + cyan countdown ring. Never skipped.
3 READ · 20–40s
Decode: givens cyan, target orange, constraints flagged, format strategy noted (ch. 04). Sting plays after this beat.
4 CONCEPT MAP · 30–90s
Concepts in play as chips (grouped by path when they differ), the misfire flagged, and top-ups — here or just-in-time mid-solve (ch. 05).
5 PATH A · core
The full method, in the step system — complete, rigorous, diagram-led. Tricks flagged in-line where they occur (ch. 08).
5b PATHS B/C · if any
Every other genuine method, compressed (ch. 07). Conditional.
6 HACK CHECK · if any
Option elimination, dimensions, limiting cases — what the question's structure gives away (ch. 09). Conditional.
7 TRAP · 30–60s
The engineered mistake, struck through in red, tied to the misfire concept from beat 4 (ch. 10).
8 LOCK-IN · 30–45s
Green answer box + 10-second confirm + path comparison table + the exam call + takeaway line. Then the end card.
Route decision.
Intro / outro:
the 2.5s intro sting plays after READ — never before the cold open — and the outro is the standard 20s end card (YouTube Guidelines, ch. 05).

02 — Question Formats · JEE Main
Card variants — Main
One card, format-aware. The badge names the exam; the format tag and stakes line change per type. Marking shown is always the actual scheme of that paper.
Single correct MCQ
JEE MAIN 2025 · JAN · S1
SINGLE CORRECT
+4 / −1 · TARGET 2:00
A particle moves so that its position vector… the angle between v and a at t = 1 s is:
(A) 30°
(B) 45°
(C) 60°
(D) 90°
Options in a row (or 2×2 when long). At LOCK-IN the correct chip fills CORRECT green with obsidian text; the trap option outlines red; the rest dim to 30%.
Single-correct MCQs always run the HACK CHECK beat — the options are information, and the video must show what they leak.
Numerical value type
JEE MAIN 2025 · APR · S2
NUMERICAL
+4 / −1 · INTEGER 0–999
…the fringe width becomes β′ = n·β/8. The value of n is ______
▮ _ _
ANSWER FIELD — cyan cursor, fills at LOCK-IN
Format tag in cyan — numerical questions are telemetry-natural. The stakes line states the answer domain (integer / rounded decimal) because it changes strategy.
No options → no elimination hacks; the HACK CHECK pivots to sanity checks: units, order of magnitude, integer-ness as a self-test ("if you didn't get an integer, you slipped").
READ addendum per format.
The READ beat always ends by naming the format's strategy in one line: MCQ — "the answer is on screen; earn it or eliminate it." Numerical — "no net below you; confirm before you commit."

Reading it right
The decode ritual is identical in every video — four passes over the question card, each with its own highlight. Students internalise the ritual by seeing it hundreds of times.

2 kg

30°

PASS 1
Givens.
Every quantity highlights cyan and simultaneously lands in a givens ledger (mono, docked right). Unstated givens — "smooth", "uniform", standard values — are added to the ledger with a "(implied)" tag.
PASS 2
Trigger words.
The phrases that select the physics — "just begins to slide", "instantaneously", "at least" — amber-dashed with a one-line gloss of what each commits you to.
PASS 3
Target.
What's actually asked, orange — with the classic mismatch called out when present ("it asks for speed, not velocity; magnitude only").
PASS 4
Format strategy.
One line on what the format offers — options to lean on, partial marks to protect, integer to self-check against (ch. 02–03).
Amber is READ-only.
Warning amber appears in this beat alone, for trigger words — it never recurs later in the video, which is what makes the decode pass visually distinct.

07 — Multiple Paths
Every genuine path
If the question has more than one honest route, the video covers them all — because the fastest route for one student is unusable for another. Paths are named, badged and compared.
PATH A · FULL METHOD
Always present, always complete, always first (unless the route-decision exception fires). Full step system, diagram-led. This is the route that works when everything else fails.
PATH B · THE ALTERNATIVE
A genuinely different method (energy instead of forces, coordinate instead of pure geometry). Compressed: 2–3 steps, same step visuals at 1.5× narrative pace, branching from the shared setup.
PATH C · THE INSIGHT
The symmetry / standard-result / clever-substitution route, when one exists. Often 30 seconds. This is usually the shareable moment — flag it for the Short.
On-screen mechanics
Path badge.
Under the docked question card: "PATH B · ENERGY METHOD". The badge is the chapter marker — description chapters list each path so students can jump to the method they want.
The fork frame.
Convergence.
Every path ends on the same answer line, shown landing in the same position — the repeated visual "different road, same city" is the proof that the methods are equivalent.

08 — Tricks & Confirms
Tricks & confirms
A
trick
is legitimate subject knowledge that accelerates a path — King's rule, symmetry, a standard result. A
confirm
is a cheap independent check of the answer. Both have fixed visual treatments so students learn to collect them.
The trick flag — inside a path
⚡ TRICK — KING'S RULE
SAVES ~1:30
The flag: ignition left-bar card, ⚡ + name, the
cue
stated as a pattern (not "here, do this"), and the time saved in cyan. It interrupts the step for ≤20s, then the step resumes.
Every trick states its
scope
: what pattern it fires on and where it breaks. A trick without its failure condition is a superstition — not shipped.
The 10-second confirm — at LOCK-IN, every video
✓ CONFIRM — LIMITING CASE
10s
θ → 0 should kill μ… tan 0 = 0 ✓. And 45° should give μ = 1 — a block on the verge at 45° is the classic boundary. The answer behaves. Commit.
The confirm menu
— pick the cheapest that applies: units/dimensions · limiting case (θ→0, m→∞, t→0) · order of magnitude · symmetry expectation · integer-ness (numerical type) · back-substitution.
Mandatory.
No DEBRIEF locks in without one confirm — "check your answer" is channel behaviour, modelled every single video, not advice.

09 — The HACK CHECK Beat
The hack check
A
hack
uses the question's own structure — its options, its units, its extremes — rather than the subject. The beat replays the question and asks:
what would a ruthless exam-taker see?
Distinct look: obsidian band, cyan accents, mono voice.
HACK CHECK — WITHOUT SOLVING
EXAM-HALL MODE
(A) 30°
(B) 45°
(C) 60°
(D) 90°
v·a = 0 is a special-angle giveaway — and at t = 1 the components are (2, 2)·(2, −2). Two options die to a 5-second dot product estimate; the survivors differ by a sign you can eyeball.
2 OPTIONS ELIMINATED · 15s · REMAINING: COIN-FLIP → SOLVE THE FAST BIT ONLY
The hack library — named, reusable, cross-referenced
H1 · OPTION ELIMINATION
Kill options by sign, parity, bounds or a known special case. Show each strike-through with its one-line reason.
H2 · DIMENSIONS & UNITS
Options with different dimensional structure → the answer falls out of [M L T] bookkeeping alone. The full dimensional table animates.
H3 · LIMITING CASES
Push a parameter to 0 or ∞ and watch which options survive. The SWEEP animation (shared with CORE) is the natural visual.
H4 · PLUG THE OPTIONS
When verifying is 10× cheaper than deriving (roots, matching values), work backwards from the options — starting with the "nicest" number.
H5 · SPECIAL VALUES
A general result asked with free parameters → choose the friendliest instance (θ = 45°, n = 2) and match options against it.
H6 · SCALE & SYMMETRY
Answer must be symmetric in a↔b, or scale as m² — options that break the symmetry are dead on sight.
Honesty rules.

10 — Trap & Lock-in
The trap & the lock-in
Beat 7 · The trap
THE TRAP
38% CHOSE (C) — PLATFORM DATA
μ = tan60°  ← resolved along the wrong axis
The misfire from beat 4, executed flawlessly — that's what makes it a trap. Option (C) exists because the examiner solved the wrong problem correctly, on purpose.
Rules unchanged from v1: one trap (the most common), strike-through in ERROR, name the option it produces, real error-rate numbers only, "the paper wants you to slip" framing — never "silly mistake".
New in v2: the trap must reference its beat-4 misfire chip by name. Concept named early + executed late = the misconception is taught twice, from both sides.
Beat 8 · Lock-in — the five fixed moves
1 · ANSWER BOX
Green surround, once per video. MCQ: correct chip fills green, trap outlines red, rest dim. Numerical: the answer field fills. Multi-correct: per-option ✓/✗ recap.
2 · CONFIRM
The 10-second confirm from the menu (ch. 08) — spoken and shown.
3 · PATH TABLE
The comparison table + the exam call (ch. 07). Skipped only when the video had a single path.
4 · TAKEAWAY
One sentence, on screen in Space Grotesk: the transferable pattern — "On the verge = friction at its ceiling. Every time."
5 · END CARD
Standard end card (YouTube Guidelines ch. 05); the linked video is the next question in the playlist or the CORE unit of the crux concept.
Solve-time telemetry.
Lock-in shows the honest clock: "Path A as shown: 3:00 · exam target: 2:30 · path C: 0:40." The gap between shown time and target time is stated, not hidden — that's the training signal.

11 — Manim Standards
Frame construction
Every frame must be flawless:
no overlapping elements, no overflowing text, no intersecting graphics, no graphics through text
. A collision anywhere is a failed render — not a note, a reject.
The zone grid — template T1, the SOLVE-beat layout (all templates: next page)
Z1 · DOCKED QUESTION — top 21%
Z2 · RAIL
Z3 · WORK AREA
Z4 · DIAGRAM DOCK
Z5 · TELEMETRY
Zones are walls.
Buffers.
≥0.4 Manim units between any two mobjects; buff≥0.25 inside cards; nothing within 48px of the frame edge (long-form) or the safe-core edge (Shorts). Leader lines route around content, never through it.
Hard rules
Text never shrinks, never spills.
If a line exceeds its zone width, split it across lines or rewrite it shorter — auto-scaling below the type floors (MathTex 40 / 56) is banned, and text touching a zone edge is a collision.
Crowding protocol.
When the work area fills: dim resolved lines to 45%, pan the camera down, or FadeOut material that's done — in that order. Never compress spacing, never stack new work over old.
Z-order.
Text always renders above graphics; graphics never draw through text or each other. Where a diagram annotation must sit on a busy region, it gets an obsidian chip behind it (92% opacity) — no naked text over strokes.
Collisions checked in code, not by eye.
The pause test.
Final QC scrubs the render at 1fps: every single frame must read as a clean, publishable diagram — mid-transition frames included. One bad frame = re-render.

11 — Manim Standards
Layout templates
The collision law is universal; the layout is not. Every beat declares its template in the storyboard — four templates cover every frame a DEBRIEF can produce. No beat ships on an undeclared layout.
T1 · SOLVE GRID
The Z1–Z5 grid (previous page). Used by PATH beats — steps, algebra, docked diagram. The workhorse, but only the workhorse.
T2 · FULL FRAME
One focal group, centred, generous margins. Used by: question cold open, PAUSE, READ, CONCEPT MAP, hack check, trap, lock-in, end card.
T3 · TOP-UP TAKEOVER
Concept explanations inside a DEBRIEF.
The solve state dims to 20% ghost — context stays visible — and the top-up card plays centre-frame, ≤30s, then the solve restores exactly as it was. A top-up is never squeezed into the work area alongside live algebra.
T4 · SHORTS STACK
9:16, inside the safe core only: question card top · work centre · caption bar bottom. No rail, no side dock — vertical scenes are composed for this stack, never cropped from T1.
One law, four layouts.

12 — The Diagram Library
One library, every diagram
All diagrams come from
nelumbo_diagrams.py
— a growing component library (FBD, incline, pulley, circuit, PV, ray, wave, vector field, graph, structure…). New diagram types are built once, reviewed, and added; ad-hoc one-off drawings are banned.
Universal diagram rules
Semantic colour, same as math:
titanium scaffold · cyan given quantities · ignition the active/target element · green confirmed results · red wrong paths. Fills only in graphite; strokes 3px (scaffold) / 4px (active).
Built, then evolved:
Create() once, early; after that the diagram only gains annotations, changes states, or morphs. Redrawing a diagram mid-video is a reject.
Labels are Mono()/Body()
at ≥28px equivalent — never LaTeX text, never rotated more than ±30°. Arrowheads from the library's single arrow style.
Diagram ↔ equation lockstep:
every term that enters an equation morphs from its diagram element. If a quantity never appears on the diagram, it doesn't belong in the diagram-led step.
Spec 01 · Free body diagrams
mg ↓ cyan · N cyan · f ORANGE (target)
CONTEXT GHOSTED 30%
The isolate move:
the body lifts out of the scene (0.6s), context dims to 30% ghost — the student watches the abstraction happen instead of receiving it done
Forces draw from the point of application, length ∝ magnitude where known
Given forces cyan; the unknown/target force ignition; components dashed titanium, drawn only when the resolution step needs them
Axes appear as a separate, ghosted layer — rotated axes (incline problems) get their own 0.4s rotation animation with the angle labelled
Each force label lands with its arrow; the equation term later morphs from that arrow (never from the label)

DEBRIEF Shorts — three cuts
One long-form DEBRIEF can spawn up to three different Shorts, each built around a different beat. All: own 1080×1920 scene, ≤60s, burned-in captions, safe-core layout, loop-friendly ending, pinned comment → long-form.
CUT 1 · THE MOVE
The key insight of PATH A (or C) in ≤3 transforms. Hook: the error rate or the clock. "41% got this 2024 integral wrong."
DEFAULT — every episode
CUT 2 · THE HACK
The hack-check beat standing alone: "solve it without solving it." Options die on screen one by one. Ends: "the full method's on the channel."
WHEN THE HACK IS CLEAN
CUT 3 · THE TRAP
The wrong path executed confidently for 15 seconds — then the strike-through. Hook: "This looks right. It's the trap." Highest-retention cut.
WHEN ERROR DATA EXISTS
Shorts rules
One beat per Short.
Never compress the nine-beat structure into 60 seconds — a Short is a single beat given the whole frame. Question card top of safe core, work centre, caption bar at safe-core bottom, mark top-left at 60%.
Release order:
Short(s) go live 24–48h after the long-form, so the pinned link lands on a live video. Cut 3 (trap) may lead a long-form by a day as a teaser when the episode is a flagship.
Selection bar:
if the beat can't stand alone without the rest of the episode, it doesn't become a Short. Zero Shorts is an acceptable outcome for an episode.
